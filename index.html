Web Workers API

Web Workers provide a simple means for web content to run scripts in background threads.

Why Web Workers?

JavaScript codebases continue to grow.

As more JavaScript runs, the more applications block ... waiting for code to finish.

Ultimately, we end up with this.

How do we write JavaScript that keeps our applications responsive?

Leveraging the Web Worker API, how do we design HTML5 application frameworks of the future?

  • Processing large arrays or JSON responses.
  • Prefetching and/or caching data for later use.
  • Analyzing video or audio data.
  • Polling of webservices.

What are Web Sockets?

Full-duplex, bi-directional communication over the Web.

But, not HTTP.

Uses a single Transmission Control Protocol (TCP) socket.

HTTP was not designed for Real Time data transfer.

With Sockets, only the data itself is sent, without the overhead of HTTP headers and cookies.

This is a game-changing, disruptive technology.

Before Web Sockets, developers had Comet hacks:

  • XHR-long polling - sequentially poll the server.
  • Reverse Ajax
  • HTTP Streaming - keep the connection open.
  • HTTP server push

These are now Polyfills for browsers without web socket support.

What is WebGL?

WebGL (Web Graphics Library) is a JavaScript API for rendering interactive 3D graphics
